# Design Tokens

This page documents the core design tokens used in the app. We use TailwindCSS with a brand color palette extended in tailwind.config.ts and default spacing/typography scales.

## Spacing

We follow Tailwind’s default spacing scale:

<div className="space-y-2 mt-3">
  {[1,2,3,4,6,8,12].map((s) => (
    <div key={s} className="flex items-center gap-3">
      <div className="w-24 text-sm text-gray-700">space-{s}</div>
      <div className={`h-4 bg-brand-200`} style={{ width: `${s * 0.25}rem` }} />
    </div>
  ))}
</div>

Use small multiples (2, 3, 4, 6, 8, 12) consistently for padding/margins.

## Typography

We use the system font stack and Tailwind font sizes:

<div className="space-y-1 mt-3">
  <div className="text-2xl font-semibold">.text-2xl / font-semibold</div>
  <div className="text-lg font-medium">.text-lg / font-medium</div>
  <div className="text-base">.text-base</div>
  <div className="text-sm text-gray-700">.text-sm / .text-gray-700</div>
  <div className="text-xs text-gray-600">.text-xs / .text-gray-600</div>
</div>

For accessibility, ensure:
- Heading levels are hierarchical (h1 → h2 → h3).
- Sufficient color contrast (brand colors pass against white for foreground text in 700+).
- Focus indicators are visible (we keep default outlines and enhance where necessary).

## Usage Notes

- Prefer brand-700 for headings/brand accents on light backgrounds.
- Use gray-50 for table headers and gray-100 backgrounds for zebra striping with even/odd rows to maintain readability.
- Avoid using brand colors for large text blocks; keep them for accents and KPIs.
